# Snap

A Goosechase-style photo scavenger hunt platform.

- `backend/` — Ruby on Rails 8 API + admin web UI (deploys to Render)
- `ios/` — SwiftUI iOS app (Snap.xcodeproj generated by XcodeGen)

## Local development

### Backend

```bash
cd backend
bin/setup        # installs gems, prepares db
bin/rails server # http://localhost:3000
```

Sign up at http://localhost:3000/signup, create a game, add teams, categories, missions.

### iOS

```bash
cd ios
xcodegen generate
open Snap.xcodeproj
```

Build & run on a Simulator. The app defaults to `http://localhost:3000` on the simulator. Tap the API URL on the auth screen to point at a custom host.

## Deploy (Render)

1. Push this repo to GitHub.
2. In Render, create a new **Blueprint** from this repo. It reads `backend/render.yaml`.
3. Set the `RAILS_MASTER_KEY` env var to the contents of `backend/config/master.key` (created locally by `bin/rails credentials:edit`).
4. (Optional) Set `S3_BUCKET`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY`, `S3_REGION`, `S3_ENDPOINT` to persist photo uploads (Cloudflare R2 / AWS S3).

After the deploy, in the iOS app point the API base URL at `https://<your-service>.onrender.com`.
